The Hilbert Curve

A single continuous line that visits every point in a square. A mathematical curiosity that powers your GPS, gene sequencing, database engines, and computer graphics — all at once.

Introduction

A line that fills all of space

Imagine you're asked to draw a path through every point inside a square — not just a grid of points, but every single point, an uncountably infinite set — using one continuous line, without lifting your pen, without crossing itself.

This sounds impossible. By classical intuition, a line is one-dimensional and a square is two-dimensional. How could a lower-dimensional object completely contain a higher one?

In 1891, mathematician David Hilbert answered this question with a construction of breathtaking elegance. By iteratively replacing a simple U-shape with smaller versions of itself, he created a curve that — in the limit — genuinely fills a two-dimensional square.

The Core Idea: At order 1, the curve is a simple U connecting 4 points. At order 2, each of the 4 segments is replaced by a smaller U, connecting 16 points. At order n, the curve visits 4ⁿ points. As n → ∞, it visits every point in the unit square — a true space-filling curve.

What makes the Hilbert curve special among all space-filling curves is its exceptional locality: points that are close together along the one-dimensional path tend to be close together in two-dimensional space, and vice versa. This property makes it extraordinarily useful in computing.

History

From a paradox to a revolution

The story of space-filling curves begins with a crisis in late 19th-century mathematics. Mathematicians had long assumed that dimension was a fundamental, inviolable property — that you couldn't map a line onto a plane in a continuous, meaningful way.

Georg Cantor shattered this assumption in 1878, proving that the real number line and the plane have the same cardinality (the same "count" of points, in the sense of infinite sets). His bijection between the two, however, was wildly discontinuous — it tore the line apart and scattered its points across the plane without any coherent structure.

The question then became: can this mapping be made continuous? Common intuition said no. Jordan curve theorems, dimension theory, and geometric reasoning all suggested that a continuous curve could not fill a 2D region.

1878
Cantor's Correspondence

Georg Cantor proves a line and a square have the same number of points (equinumerosity), shocking the mathematical world. But his bijection was not continuous.

1890
Peano's First Curve

Giuseppe Peano constructs the first continuous surjective map from [0,1] onto [0,1]². He described it analytically, but left no picture. It immediately inspired Hilbert.

1891
Hilbert's Geometric Construction

David Hilbert publishes a simple, visual, geometric construction. With just four recursive rules and a picture at each stage, he made space-filling curves accessible and intuitive.

1904–1914
Theory Develops

Mathematicians prove dimension theory rigorously (Brouwer, Lebesgue). Despite Peano's curve, dimension is still well-defined — these curves are surjective but not injective (they self-intersect in the limit).

1969–present
Computational Rebirth

Morton (1966) and others discover that space-filling curve orderings dramatically improve cache performance and spatial query efficiency. Hilbert's mathematical curiosity becomes an engineering workhorse.

Mathematics

The rigorous beauty underneath

1. Formal Definition

The Hilbert curve is defined as the limit of a sequence of piecewise-linear curves {H₁, H₂, H₃, …}. Each Hₙ is a continuous, non-self-intersecting path through 4ⁿ points arranged in a 2ⁿ × 2ⁿ grid.

// Each iteration is a continuous function:
Hn : [0, 1] → [0, 1]²

// The limit curve H is defined as:
H = limn→∞ Hn

// This limit exists because the sequence is uniformly Cauchy.
// ‖H_{n+1}(t) − H_n(t)‖ ≤ √2 / 2ⁿ for all t ∈ [0,1]

The key theorem: H is continuous, surjective, and measure-preserving. It maps the unit interval [0,1] onto the entire unit square [0,1]², touching every point (continuity + surjectivity), and it maps uniform probability on [0,1] to uniform area measure on the square (measure-preservation).

2. Recursive Construction

The construction uses four orientations of a U-shaped connector. At each stage, the square is divided into four quadrants, and the previous curve is placed in each quadrant — sometimes rotated or reflected — and connected.

// Recursive drawing function:
hilbert(x, y, ax, ay, bx, by, order):
  // ax,ay = x-axis vector bx,by = y-axis vector
  if order == 0:
    plot(x + (ax + bx)/2, y + (ay + by)/2)
  else:
    hilbert(x, y, by/2, ay/2, bx/2, ay/2, order−1)
    hilbert(x+ax/2, y+ay/2, ax/2, ay/2, bx/2, by/2, order−1)
    hilbert(x+ax/2+bx/2, y+ay/2+by/2, ax/2, ay/2, bx/2, by/2, order−1)
    hilbert(x+ax/2+bx, y+ay/2+by, −by/2,−ay/2, −ax/2,−ay/2, order−1)

3. L-System Grammar

The Hilbert curve can be elegantly expressed as a Lindenmayer system (L-system), a formal grammar that rewrites symbols according to production rules and then interprets the final string as drawing instructions.

Axiom: A
A → −BF+AFA+FB−
B → +AF−BFB−FA+

F = move forward  |  + = turn left 90°  |  − = turn right 90°
A, B = recursive sub-structures (no drawing action)

Applying these rules n times produces a string that, when interpreted by a turtle graphics system, draws the order-n Hilbert curve exactly. This reveals the deep self-similar structure: A and B are just rotated/reflected copies of each other.

4. Coordinate Algorithms

The power of the Hilbert curve in computing comes from efficient algorithms to convert between 1D position (distance along the curve) and 2D coordinates:

d → (x, y) · Decode

function d2xy(n, d) {
  let x = 0, y = 0;
  let rx, ry, t = d;
  for (let s = 1; s < n; s <<= 1) {
    rx = 1 & (t >> 1);
    ry = 1 & (t ^ rx);
    if (ry === 0) {
      if (rx === 1) {
        x = s−1−x;
        y = s−1−y;
      }
      [x, y] = [y, x];
    }
    x += s * rx;
    y += s * ry;
    t >>= 2;
  }
  return [x, y];
}

(x, y) → d · Encode

function xy2d(n, x, y) {
  let d = 0, rx, ry;
  for (let s = n>>1; s > 0; s >>= 1) {
    rx = (x & s) > 0 ? 1 : 0;
    ry = (y & s) > 0 ? 1 : 0;
    d += s * s * ((3 * rx) ^ ry);
    // rotate quadrant
    if (ry === 0) {
      if (rx === 1) {
        x = s−1−x;
        y = s−1−y;
      }
      [x, y] = [y, x];
    }
  }
  return d;
}

Both algorithms run in O(log n) time where n = 2order. The bit-manipulation approach makes them extraordinarily fast on modern hardware — a single CPU instruction can often handle the rotation step.

5. Fractal Dimension

The Hilbert curve is a fractal. Its Hausdorff (fractal) dimension is exactly D = 2, equal to the dimension of the plane it fills. This is computed by the scaling relationship:

D = log(N) / log(r)   // where N = number of self-similar copies, r = scaling factor

At each step: N = 4 copies, each scaled by r = 2
D = log(4) / log(2) = 2 / 1 = 2 ✓

// Unlike a regular curve (D = 1), the Hilbert curve's infinite complexity
// collapses its dimension up to 2 — matching the 2D space it inhabits.

6. Locality Property

The most important computational property: points that are nearby in 1D are likely to be nearby in 2D, and nearby 2D points tend to map to nearby 1D positions.

Formal locality: For any two points p, q on the Hilbert curve with |H⁻¹(p) − H⁻¹(q)| ≤ ε (nearby on the curve), the maximum possible |p − q| in 2D is bounded. The Hilbert curve achieves the best possible locality among all space-filling curves — it minimizes the worst-case 2D distance for nearby 1D positions.
Key Properties

What makes it extraordinary

∞
Space-Filling
In the limit (n→∞), the curve passes through every single point in the unit square. It is surjective: the image of H equals [0,1]².
〰
Continuity
H is uniformly continuous — small changes in the 1D parameter t produce small changes in 2D position. No jumps, no teleportation.
🔁
Self-Similarity
Each quarter of the Hilbert curve is a scaled, rotated/reflected copy of the whole curve. This fractal property makes it recursively constructable.
📐
Measure-Preserving
Uniform measure on [0,1] maps to uniform Lebesgue measure on [0,1]². Every subregion of the square gets a proportional share of the curve.
📍
Locality-Optimal
Among all space-filling curves, the Hilbert curve achieves the best possible locality — minimizing the worst-case spatial distance between nearby points on the curve.
🔀
Not Injective
H is not one-to-one — it "touches" boundary points multiple times. Every point inside the square has exactly one preimage; boundary points can have up to 3.
📈
Infinite Length
The length of Hₙ is (4ⁿ − 1)/3^(n−1) → ∞. The limit curve has infinite one-dimensional length, consistent with filling a 2D area.
🔢
Nowhere Differentiable
Like most fractals, the Hilbert curve has no tangent anywhere — it changes direction infinitely often, making it nowhere differentiable.
Applications

How it solves real problems

The Hilbert curve is not an abstract curiosity. Its locality-preserving property translates directly into measurable performance improvements across an extraordinary range of fields. Here's how it works in each domain.

Computer Science
Spatial Database Indexing

Modern spatial databases must answer queries like "find all restaurants within 5 km of this GPS coordinate." Storing rows by latitude/longitude separately means a range query touches vast swaths of unrelated data. You need a way to assign each 2D location a single 1D key that preserves spatial relationships.

The Problem
A naïve row scan to find all points inside a bounding box on a billion-row table would take hours. Simple Morton (Z-order) codes work but have clustering artifacts at quadrant boundaries.
Hilbert's Solution
Encode each (lat, lon) as a Hilbert index. Sort rows by this index. Now a spatial range query becomes a small number of contiguous 1D range scans — typically 10–30× faster than alternatives. PostgreSQL, Google BigQuery, and Apache Parquet all offer Hilbert-based spatial indexing.

Why Hilbert beats Z-order: Z-order curves have "jumps" at quadrant boundaries — two nearby points can have very different Z-codes. The Hilbert curve, by always connecting the four sub-quadrants in a U-shape, eliminates these pathological boundary cases and maintains better average locality.

Image Processing & Graphics
Image Compression & Texture Mapping

JPEG compression divides images into 8×8 pixel blocks and applies DCT transforms. But the order in which pixels are encoded within each block, and how blocks are stored in memory, dramatically affects compression ratios and cache performance.

The Problem
Raster scan order (left-to-right, top-to-bottom) has poor locality for 2D image data. When a GPU renders a texture, a diagonal line of pixels causes constant cache misses because nearby screen pixels are far apart in memory.
Hilbert's Solution
Hilbert-ordering pixels ensures spatially adjacent pixels are also adjacent in memory. Modern GPU texture atlases use Hilbert-like Morton/Hilbert tiling to achieve near-optimal cache hit rates. Game engines see 20–40% speedups in texture sampling for complex scenes.

Dithering: Hilbert-curve dithering for error diffusion (reducing a full-color image to fewer colors) produces visually superior results to raster-scan dithering because errors are spread in a space-filling pattern rather than in scanlines.

Bioinformatics & Genomics
Hi-C Genome Folding & Sequence Visualization

The human genome is 3 billion base pairs long — but it's not a straight line. DNA folds into complex 3D structures inside the cell nucleus, and understanding these folding patterns (chromatin organization) is critical for understanding gene regulation and disease.

The Problem
Hi-C experiments produce massive contact matrices: a 3Gb × 3Gb matrix of interaction frequencies between all pairs of genomic loci. Visualizing, storing, and querying this data is computationally intractable with conventional methods.
Hilbert's Solution
Map genomic position to a Hilbert curve index. The 1D genome sequence naturally embeds into 2D Hilbert space, revealing chromosomal territories, topologically associating domains (TADs), and A/B compartments. Tools like HiGlass use Hilbert-indexed tile pyramids for interactive genome browsing at any scale.

Bonus: Hilbert curve genome visualization also enables comparison of two genomes — syntenic regions (conserved blocks) appear as diagonal stripes, intuitively revealing evolutionary relationships.

Computer Architecture
Cache-Oblivious Matrix Algorithms

Matrix multiplication is one of the most performance-critical operations in computing — it underlies all neural networks, scientific simulations, and graphics. The naive triple-loop algorithm is limited by memory bandwidth, not CPU speed.

The Problem
Row-major storage of matrices causes cache thrashing during multiplication: multiplying an M×K matrix by a K×N matrix accesses one matrix by rows and the other by columns — one access pattern is always cache-hostile.
Hilbert's Solution
Hilbert-curve matrix layout stores matrix elements so that any 2D subregion is contiguous in memory. Cache-oblivious algorithms recurse on submatrices — the Hilbert layout ensures each sub-problem fits optimally in cache. This achieves O(n³ / B√M) cache complexity (optimal) vs O(n³ / B) for naive layouts.

Libraries like BLAS/LAPACK, NumPy, and PyTorch use tiled matrix layouts inspired by space-filling curves to achieve near-peak memory bandwidth utilization.

Networking & Internet Infrastructure
IPv4/IPv6 Address Space Visualization

The IPv4 address space contains 2³² ≈ 4.3 billion addresses. Understanding which address ranges belong to which countries, ISPs, and organizations is essential for network security, routing policy, and internet governance.

The Problem
A linear address space plotted linearly shows nothing useful. IP address allocations are hierarchical and clustered, but a 1D plot makes geographic and topological clustering invisible.
Hilbert's Solution
Map the 2³² IPv4 address space onto a 65536×65536 Hilbert curve grid. CAIDA's internet mapping project uses this to visualize global IP allocation — /8 blocks appear as coherent 2D regions, and geographically co-located addresses cluster visually. BGP routing anomalies become instantly visible as spatial outliers.

The same technique works for IPv6 (2¹²⁸ addresses). Hilbert visualization is now standard in network security dashboards for detecting DDoS attacks, port scans, and botnets — anomalous traffic shows up as unexpected spatial clusters.

Scientific Computing
N-Body Simulations & Domain Decomposition

Simulating the gravitational interactions of millions of particles (stars in a galaxy, atoms in a protein, fluid vortices) is one of the most computationally intensive tasks in science. These simulations must be parallelized across thousands of CPU cores.

The Problem
Distributing particles across processors naïvely (by coordinate ranges) creates severe load imbalance — denser regions overwhelm some processors while others sit idle. Communication between processors for nearby particles creates bandwidth bottlenecks.
Hilbert's Solution
Sort all particles by their Hilbert index, then assign contiguous segments to processors. This guarantees each processor gets spatially cohesive regions (minimizing communication) and can be load-balanced by adjusting segment lengths. The GADGET cosmological simulation code uses this for galaxy formation at supercomputer scale.
Psychology & Perception

Why the brain loves this curve

The Hilbert curve's appeal isn't purely intellectual. Humans respond to it on a deep perceptual and aesthetic level — and science is beginning to understand why.

01
The Fractal Preference
Psychologist Richard Taylor's studies showed humans consistently prefer fractal patterns with dimension D ≈ 1.3–1.5 — the range found in coastlines, trees, and clouds. But Hilbert curves (D=2) still fascinate us, perhaps because we perceive each finite iteration (D close to 1.5) rather than the limit. Eye-tracking studies show viewers explore Hilbert curves in smooth, guided sweeps — the path provides a natural visual scaffold.
02
Predictable Complexity
The brain seeks patterns. Pure randomness is stressful; pure order is boring. Fractal structures hit a sweet spot: locally complex, globally structured. The Hilbert curve is maximally complex (space-filling) yet obeys strict rules at every scale. This creates what psychologists call aesthetic pleasure from pattern recognition — the "aha" moment when the recursive rule clicks.
03
Gestalt Completeness
Gestalt psychology describes our drive to perceive wholes. The Hilbert curve satisfies this powerfully: the finite approximations feel "complete" (they fill a square perfectly), while simultaneously implying infinite detail. This triggers the closure principle — we perceive a finished, bounded, satisfying form even though the underlying mathematical object is infinite.
04
Scale Invariance & Awe
Psychologist Dacher Keltner's research on awe shows it's triggered by vastness and things that exceed existing mental schemas. The Hilbert curve triggers mild awe: zoom in, and you see the same structure repeating forever. This self-similarity echoes natural phenomena (fern fronds, river deltas, bronchi) that evolution has primed us to find meaningful and navigable.
05
Data Visualization Advantage
When complex multi-dimensional data is mapped onto a Hilbert curve and then colored, the human visual system can identify clusters, outliers, and gradients far faster than with tabular or scatter displays. Studies of genome browsers show scientists detect structural variants up to 3× faster when chromosomal contacts are Hilbert-displayed versus traditional heatmaps. Our brains are spatial processors — the Hilbert curve gives spatial structure to non-spatial data.
06
Mathematical Beauty
Semir Zeki's neuroaesthetics research shows that understanding a beautiful mathematical proof activates the same medial orbito-frontal cortex region as experiencing beautiful music or art. The Hilbert curve combines visual beauty with intellectual elegance — a simple 4-case recursive rule produces infinite complexity. This double satisfaction (sensory + intellectual) makes it one of the most beloved objects in mathematics.
